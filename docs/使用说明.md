# 使用说明

本文档用于说明如何理解、部署和验证这套酒馆插件 + 网页前端。

当前项目定位为 `astrbot_plugin_tavern_lobby`。它不是 SillyTavern 角色卡解析 / 转换插件。

## 1. 整体关系

整套系统由两部分组成：

### A. AstrBot 插件

负责：

- 在群里响应命令
- 维护房间状态
- 分配身份
- 推进回合
- 把公开状态同步给网页服务

源码位置：`astrbot_plugin/main.py`

### B. tavern-web

负责：

- 提供 `room` 参与者房间页
- 提供 `spectator` 旁观页
- 存储公开时间线与参与者聊天
- 通过 WebSocket 实时更新页面

源码位置：`tavern-web/`

---

## 2. 基本使用流程

### 群里流程

1. 房主发送：`@小预 酒馆启动`
2. 房主可发送：`@小预 剧本列表`
3. 房主发送：`@小预 选择剧本 剧本名`
4. 其他玩家发送：`@小预 加入酒馆`
5. 人数满足后，房主发送：`@小预 开始游戏`
6. 玩家发送：`@小预 我的身份`
7. 房主发送：`@小预 下一回合` 推进阶段

### 网页流程

- 房间创建时会给出：
  - 房间页链接
  - 旁观页链接
- 玩家进入 `room` 页可发送网页聊天
- 旁观者进入 `spectator` 页只看公开实况

---

## 3. 当前关键行为

### 身份私发

`我的身份` 当前按私发处理：

- 成功：群里提示已私发
- 失败：群里提示失败，但不会公开身份

### 链接只出现一次

当前逻辑中：

- 只有创建酒馆时会发房间链接和旁观链接
- 后续加入、开始、状态、推进回合等群消息不再重复发链接

### room 页聊天布局

当前网页消息布局为：

- 自己的发言：右侧
- 别人的发言：左侧
- 主持人公开播报：系统消息卡片

## 4. 本地运行 tavern-web

```bash
cd tavern-web
npm install
npm run start
```

默认地址：

- `http://127.0.0.1:8088`

如果需要改端口，可设置：

```bash
TAVERN_WEB_PORT=8090
```

---

## 5. 验证方式

### 本地验证

```bash
cd tavern-web
npm run verify:phase1
npm run verify:ws
npm run verify:chat
npm run verify:ui
```

### 远端验证

如果已经部署到服务器，可在远端目录执行：

```bash
cd /opt/tavern-web
node verify_phase1.mjs
node verify_ws_e2e.mjs
node verify_chat_e2e.mjs
node verify_ui_e2e.mjs
```

### AstrBot 插件验证

```bash
docker exec astrbot python -m py_compile /AstrBot/data/plugins/astrbot_plugin_tavern_lobby/main.py
docker restart astrbot
```

然后检查日志中是否存在：

- `error`
- `exception`
- `traceback`

## 6. 推荐部署思路

### AstrBot 插件

将 `astrbot_plugin/main.py` 作为主逻辑参考，放到实际 AstrBot 插件目录中。

### tavern-web

将 `tavern-web/` 作为独立 Node.js 服务部署。

推荐保留：

- `src/`
- `web/`
- `package.json`
- `server.js`
- 验证脚本

不建议把以下内容带入生产：

- 本地测试数据库
- `node_modules`
- 临时备份
- 归档 tar 包

---

## 7. 已知边界

### 无账号模型

当前网页侧没有账号系统，因此“我的消息”识别依据是：

- 浏览器本地保存的网页昵称

这意味着相同昵称在同一浏览器模型下会共用“自己消息”的判断。

### 插件发布结构仍可继续标准化

当前仓库已经能表达真实源码，但若后续打算公开发布为标准插件，仍建议补：

- 插件安装元数据
- 配置模板
- 部署脚本
- 版本说明
