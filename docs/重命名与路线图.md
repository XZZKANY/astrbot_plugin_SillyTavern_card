# 重命名与路线图

## 1. 项目定位结论

当前项目应定位为：

```text
astrbot_plugin_tavern_lobby
```

而不是：

```text
astrbot_plugin_SillyTavern_card
```

原因：

- 当前实际功能是群聊酒馆组队、剧本流程、身份发放、网页房间同步
- 当前没有 SillyTavern PNG / JSON / WEBP 角色卡解析能力
- 继续使用 `SillyTavern_card` 名称会误导用户和插件生态

## 2. 推荐仓库动作

### 必做

1. 在 GitHub 仓库 Settings 中将仓库名改为：

```text
astrbot_plugin_tavern_lobby
```

2. 本地更新 remote：

```bash
git remote set-url origin https://github.com/XZZKANY/astrbot_plugin_tavern_lobby.git
```

3. 保留旧仓库跳转即可，GitHub 通常会自动转发旧地址。

### 暂不建议

不建议继续在当前项目里补 SillyTavern card 解析功能。

如果要做角色卡工具，应另开仓库，例如：

```text
astrbot_plugin_sillytavern_card_converter
```

## 3. P0 路线图

### 3.1 发布结构补齐

当前已补：

- `metadata.yaml`
- `requirements.txt`
- `tavern_web_config.example.json`
- `tavern-web/.env.example`
- 根 README 与分目录 README

后续建议继续补：

- LICENSE
- Dockerfile
- docker-compose.yml
- systemd 示例
- GitHub Actions CI
- release notes

### 3.2 安全边界补齐

当前 room / spectator 在前端和接口层已经分离，但还不是完整权限模型。

下一步建议：

- 为参与者链接签发 participant token
- 为旁观者链接签发 spectator token
- `/api/rooms/:roomId/room-view` 必须验证 participant token
- `/api/rooms/:roomId/chat` 必须验证 participant token
- WebSocket subscribe 必须验证 token

### 3.3 主文件拆分

当前 `astrbot_plugin/main.py` 仍然承担过多职责。

建议拆分为：

```text
astrbot_plugin/
├─ main.py                  # AstrBot adapter
├─ domain/
│  ├─ room.py
│  ├─ script.py
│  ├─ role_assignment.py
│  └─ state_machine.py
├─ application/
│  ├─ commands.py
│  └─ room_service.py
├─ infrastructure/
│  ├─ state_store.py
│  └─ web_sync_client.py
└─ content/
   └─ scripts/*.yaml
```

### 3.4 剧本内容外置

当前 `SCRIPT_LIBRARY` 硬编码在 Python 中。

建议迁移为：

```text
content/scripts/campus_horror.yaml
content/scripts/fantasy_adventure.yaml
```

并补 schema 校验。

## 4. P1 路线图

- Bot 到 Web 同步改为异步 HTTP 客户端
- WebSocket broker 增加鉴权、心跳和重连语义
- API 输入改为 schema validation
- 增加房主控制台
- 增加 `/酒馆链接` 或链接私发命令，避免“只发一次”后新玩家找不到链接

## 5. P2 路线图

- 剧本市场 / 剧本包导入
- 房间复盘导出
- 多平台适配
- 托管版 tavern-web
- 活动数据统计

## 6. 当前结论

当前仓库应先完成改名和发布结构补齐，不应继续以 SillyTavern card 名称添加酒馆功能。
